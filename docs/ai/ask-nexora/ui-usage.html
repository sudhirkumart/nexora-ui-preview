Dummy data only

Do not use a production tenant, production credentials, beneficiary data, donor data, real NGO or church documents, customer information, exports, support records, emails, or operational documents. Create purpose-built synthetic files for this POC.

Local prerequisites

  • The existing NEXORA backend is running locally on http://127.0.0.1:8000.
  • The RAG foundation migration and synthetic seed data have been applied to a disposable local test database.
  • ASK_NEXORA_RAG_POC_ENABLED=true is set only for that controlled local backend process.
  • An approved synthetic test user exists. Do not add credentials to this repository, source files, screenshots, or documentation.

The backend repository's AI data-handling policy, RAG POC scope, progress log, and RAG usage guide remain the authority for backend setup and tenant-isolation validation.

Start the UI

From the nexora-ui-preview repository:

npm install
npm run dev

Vite serves the preview, normally at http://localhost:5173, and proxies same-origin /api requests to the local backend on port 8000.

Review workflow

1. Authenticate

Open /ask-nexora. The protected route redirects to /ask-nexora/login. Sign in with an approved synthetic local account.

2. Select a dummy file

Choose a UTF-8 .txt, .md, .markdown, .csv, or .json file created specifically for testing.

3. Confirm dummy data

Read and select the confirmation checkbox. The UI sends dummy_data_confirmed=true only after this explicit confirmation.

4. Upload

The backend validates the file, creates deterministic local chunks, and returns the document status and chunk count.

5. Ask a question

Ask a concise question supported by the uploaded dummy file. The UI scopes the query to the returned document ID.

6. Review evidence

Confirm the response shows local_dummy, the answer status, ranked source labels, excerpts, and reference IDs.

Fail-closed behavior

A 401 response ends the UI session and returns the user to sign-in. A 403 response places the page in an unavailable state because the backend feature flag or permissions denied access. The UI does not contain a client-side override.

  • Unsupported, empty, oversized, malformed, or non-UTF-8 files display the backend validation error.
  • The question field remains disabled until a dummy document is uploaded successfully in the current UI session.
  • An answer with a mode other than local_dummy is not displayed by this POC UI.
  • Insufficient evidence may return no sources and should not be treated as an authoritative answer.

Security properties

The UI adds no API keys, provider endpoints, external LLM calls, embeddings, vector database, telemetry, tracing, analytics, connectors, background jobs, autonomous actions, or writes to NEXORA business data.

  • The access token is held in browser sessionStorage and is cleared on sign-out or when the tab session ends. The refresh token returned by the backend is not stored.
  • Tenant identity is never accepted from a UI field or request body. The backend derives it from the authenticated user and enforces tenant scope before retrieval.
  • Uploaded dummy files and POC answers may be persisted by the existing tenant-scoped backend and its internal audit controls.
  • All API requests are relative, same-origin browser requests through the local Vite proxy.

Known limitations

  • The backend exposes no feature-status endpoint, so availability is confirmed only when upload or query is attempted.
  • The backend exposes no document-list endpoint, so the UI works with one document uploaded during the current page session.
  • The static Vercel preview does not include a backend proxy. Ask NEXORA is local-only until an approved same-origin deployment architecture exists.
  • This is not a production authentication implementation, document library, AI assistant, or production Ask NEXORA release.
